LAYOUT: FIXED LEFT SIDEBAR + MAIN CONTENT AREA

The app uses a two-column layout at all times:
- Left: fixed sidebar navigation (width: ~240px), always visible
- Right: main content area, scrollable

The sidebar contains exactly 4 navigation items (numbered, icon + label):
  1. Domainanalyse – Basisdaten
  2. HTML-Analyse
  3. Ergebnisse
  4. Einstellungen

Active item is highlighted. Clicking any item switches the main content 
area to that view — no page reloads, pure client-side routing (React state).

After an analysis has completed, the user can click items 1 or 2 at any 
time to modify inputs and re-trigger a full new analysis. Do not lock 
navigation after analysis runs.

  platform name as label, input as URL or handle):
    LinkedIn, Facebook, Instagram, YouTube, TikTok,

  - Textarea, ~4 rows

  Label: "Analyse starten"

  Label: "Analyse starten"

SIDEBAR ITEM 3 — "Ergebnisse"

This view has two states:

STATE A — Analysis running:
  Show a vertical list of all analysis modules with status indicators:
    ⏳ waiting  /  🔄 running (animated spinner)  /  ✅ done  /  ⚠️ skipped
  Show current module name + a subtle progress bar at the top.
  Modules run sequentially; update status in real time via 
  server-sent events (SSE) or WebSocket from the FastAPI backend.

STATE B — Analysis complete:
  Display the full report (all modules, charts, recommendations).
  At the top of the results: a "Analyse herunterladen" button that 
  generates and downloads a self-contained HTML report file 
  (single file, inline CSS + SVG charts, no external dependencies).
  
  Below the download button: a subtle text prompt:
  "Basisdaten ändern oder neue Analyse starten? 
   → Wechseln Sie zu Domainanalyse oder HTML-Analyse."
  (This is purely informational — navigation is always available 
  in the sidebar.)

If no analysis has been run yet (first app load), show a neutral 
empty state: icon + text "Noch keine Analyse durchgeführt. 
Starten Sie unter Domainanalyse oder HTML-Analyse."

SECTION: Darstellung
  Label: "Farbmodus"
  Three-option radio/toggle group:
    ○ Hell
    ○ Dunkel  
    ● Systemeinstellung (default, selected on first load)

  light and dark themes. Do not use two separate CSS files.

GENERAL UI REQUIREMENTS

- All UI copy (labels, tooltips, placeholders, buttons, status messages) 
  in German.
- Sidebar is always visible and never collapses (desktop layout only 
  is fine for v1 — no mobile breakpoint required).
- Tooltips: implement as native HTML title attributes OR as small 
  ⓘ icon next to the label that shows a styled tooltip on hover. 
  Do not use a tooltip library — implement in plain CSS/React.
- Form state (all inputs in items 1 and 2) must persist in React 
  global state (Context or Zustand) so that switching between sidebar 
  items does not clear entered data.
- Do not use a multi-step wizard or modal overlays for any input. 
  All forms are inline in the main content area.
