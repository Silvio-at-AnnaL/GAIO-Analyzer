BACKEND: load prompts at runtime

Create server/prompt-manager.ts:

  import { query } from './db'
  
  // Cache prompts in memory after first load
  const cache = new Map<string, string>()
  
  export async function getPrompt(
    slug: string
  ): Promise<string> {
    // Check cache first
    if (cache.has(slug)) {
      return cache.get(slug)!
    }
    
    const result = await query(
      'SELECT template FROM prompts 
       WHERE slug = $1',
      [slug]
    )
    
    if (result.rows.length === 0) {
      throw new Error(
        `Prompt nicht gefunden: ${slug}`
      )
    }
    
    const template = result.rows[0].template
    cache.set(slug, template)
    return template
  }
  
  export function clearPromptCache(
    slug?: string
  ) {
    if (slug) {
      cache.delete(slug)
    } else {
      cache.clear()
    }
  }
  
  export function fillTemplate(
    template: string,
    vars: Record<string, string>
  ): string {
    return Object.entries(vars).reduce(
      (t, [key, val]) => 
        t.replaceAll(`{{${key}}}`, val),
      template
    )
  }

In every analysis module that currently 
has a hardcoded prompt string:
  1. Replace the inline string with:
       const template = await getPrompt('slug')
       const prompt = fillTemplate(template, {
         CRAWLED_CONTENT: content,
         COMPANY_NAME: name,
         // etc.
       })
  2. Convert the prompt variable names 
     to {{PLACEHOLDER}} format in the 
     seeded template.

When an admin saves an edited prompt, 
call clearPromptCache(slug) so the next 
analysis loads the updated version.

BACKEND: API routes

  GET /api/admin/prompts
    Admin only.
    Returns all prompts: id, slug, name, 
    description, module, is_modified, 
    updated_at.
    Does NOT return template text 
    (keep response small for the list view).
    Grouped by module in the response.

  GET /api/admin/prompts/:slug
    Admin only.
    Returns full prompt including template 
    and placeholders JSON.

  PATCH /api/admin/prompts/:slug
    Admin only.
    Body: { template: string }
    Validates template is not empty.
    Updates template, sets is_modified=true,
    updates updated_at.
    Calls clearPromptCache(slug).
    Returns updated prompt.

  POST /api/admin/prompts/:slug/reset
    Admin only.
    Resets template to the hardcoded 
    default value for that slug.
    Sets is_modified=false.
    Calls clearPromptCache(slug).
    
    The hardcoded defaults must be stored 
    as a constant in the backend 
    (server/prompt-defaults.ts) so reset 
    always has access to the original text.
