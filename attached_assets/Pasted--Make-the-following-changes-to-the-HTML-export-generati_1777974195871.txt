B — PDF: Radar chart position fix

The radar chart (Dimensionen / Netzdiagramm) 
has been shifting to the right in the PDF 
since the page height calculation changes.

This is caused by the chart's SVG or container 
not being re-centered after the layout 
changes made during PDF capture preparation.

Find the radar chart component in the results 
header and ensure its container has:
  - width: 100% (not a fixed pixel width)
  - text-align: center or margin: 0 auto
  - overflow: visible (not hidden)

Before capturing the results header for PDF, 
add this to the header element temporarily:
  headerEl.style.overflow = 'visible'

Also ensure the chart container inside the 
header has no absolute or fixed positioning 
that could cause drift when the surrounding 
layout changes.

If the chart uses a ResponsiveContainer from 
Recharts (or similar), ensure its parent div 
has an explicit pixel width set during capture:
  chartContainer.style.width = '500px'
  chartContainer.style.minWidth = '500px'
Set this before the animation frame wait and 
restore it after capture.
