Fix five issues in the Angebots-Creator 
(AngebotCreatorView + backend endpoint).

---

FIX 1 — Scores show as "–/100" instead of 
actual values

The AI prompt receives the scores but they 
are not being interpolated into the prompt 
string. The scores_json field from the 
analysis_log is stored as a JSON string — 
it must be parsed before use.

In the backend endpoint 
POST /api/admin/angebot/generate:

  const scoresRaw = analysisRow.scores_json
  const scores = scoresRaw 
    ? JSON.parse(scoresRaw) 
    : {}

Then in the prompt, replace placeholder 
references with actual values:
  
  `GAIO Gesamtscore: ${analysisRow.gaio_score}/100
  Technisches SEO: ${scores.technical ?? '–'}/100
  Schema.org: ${scores.schema ?? '–'}/100
  Heading-Struktur: ${scores.headings ?? '–'}/100
  Inhaltliche Relevanz: ${scores.content ?? '–'}/100
  FAQ-Qualität: ${scores.faq ?? '–'}/100
  LLM-Auffindbarkeit: ${scores.llm ?? '–'}/100`

Check the actual field names in scores_json — 
they may be camelCase or snake_case depending 
on how the analysis stores them. Log the 
parsed scores object to console to verify 
the exact field names, then use those.

---

FIX 2 — Text gets cut off mid-sentence

The callLLM() function is called with a 
max_tokens value that is too low for a full 
proposal (the default of 1000 tokens is 
far too short for a multi-section offer).

In the angebot generate endpoint, call 
callLLM with a higher token limit:

  const result = await callLLM(prompt, 4000)

If callLLM does not accept a maxTokens 
parameter, find where the Anthropic/OpenAI 
call is made and increase max_tokens to 
4000 for this specific endpoint.

Also add this instruction at the end of 
the prompt to prevent truncation:

  "Wichtig: Schreibe das Angebot vollständig 
  zu Ende. Brich niemals mitten in einem 
  Satz oder Abschnitt ab. Der letzte 
  Abschnitt muss mit einem vollständigen 
  Schlusssatz enden."

FIX 3 — Remove ```html wrapper from output

The AI returns the HTML wrapped in a 
markdown code fence (```html ... ```). 
Strip this before passing to the editor.

In the backend, after receiving the AI 
response, clean the output:

  let html = rawResponse.trim()
  
  // Remove opening ```html or ``` fence:
  html = html.replace(/^```html\s*/i, '')
  html = html.replace(/^```\s*/i, '')
  
  // Remove closing ``` fence:
  html = html.replace(/\s*```\s*$/, '')
  
  html = html.trim()

Also add this instruction to the prompt:

  "Antworte ausschließlich mit dem reinen 
  HTML-Inhalt. Keine Markdown-Code-Fences, 
  kein ```html, keine Erklärungen davor 
  oder danach. Beginne direkt mit <h1>."

---

FIX 4 — Add <br> after <hr> tags

The Preisrechner renders content directly 
after <hr> without spacing. Add a <br> 
after every <hr> in the generated output.

In the backend cleanup step (after 
stripping the markdown fence):

  html = html.replace(/<hr\s*\/?>/gi, 
    '<hr><br>')

Also add this instruction to the prompt:

  "Füge nach jedem <hr>-Tag immer ein 
  <br>-Tag ein."
