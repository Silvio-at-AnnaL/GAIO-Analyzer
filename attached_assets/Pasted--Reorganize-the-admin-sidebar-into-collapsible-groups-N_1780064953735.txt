SIDEBAR BEHAVIOR

In Sidebar.tsx, replace the current flat 
list of admin items with grouped rendering.

For each group in ADMIN_NAV_GROUPS:

  VISIBILITY RULE:
    A group is rendered if and only if 
    AT LEAST ONE of its items passes 
    canAccess(itemId, userRole, permissions).
    If none of the items in a group are 
    accessible, the group heading is 
    not shown at all.

  EXPAND/COLLAPSE STATE:
    Each group has its own open/closed 
    state: useState<boolean>
    
    Default state on load:
      Open if any sub-item matches the 
      currently active view.
      Closed otherwise.
    
    State persists during the session 
    (in component state, not localStorage).

  GROUP HEADER rendering:
    A clickable row that toggles open/closed.
    
    Layout (flex row):
      [Icon 15px]  [Label text-sm font-medium]  
      [ChevronDown / ChevronRight 14px, right-aligned]
    
    Styling:
      Same text color as regular nav items
      Slightly bolder than sub-items 
        (font-weight: 600)
      No active/highlight state on the 
        group header itself — it is never 
        "active", only its children are
      Hover: subtle background tint
      Cursor: pointer
      Padding: same horizontal as other 
        nav items, slightly less vertical
    
    The chevron rotates 90° when open 
    (CSS transition, 150ms).

  SUB-ITEMS rendering (when group is open):
    Only rendered if canAccess() returns true 
    for that item.
    
    Indented relative to group header:
      padding-left: 28px (or use pl-7)
    
    Slightly smaller text than group header:
      text-sm (same size) but font-normal
    
    No icon — the sub-items are text only.
    Active state: same blue left-border 
    highlight as current nav items.
    
    Animated: sub-items slide in/out 
    when group toggles (max-height 
    transition or Tailwind's group 
    approach).

  AUTO-EXPAND:
    When the user navigates to a sub-item 
    (e.g. by clicking a link elsewhere in 
    the app that sets activeView), 
    automatically open the parent group 
    if it is closed.
