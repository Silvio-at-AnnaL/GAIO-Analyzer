Build a complete admin area for the GAIO Analyzer.
This is a significant new feature — implement it 
carefully and completely. Backend and frontend.

---

TECH STACK DECISIONS

Database: Use SQLite with better-sqlite3 
(lightweight, no separate server needed, 
already likely available in the project).
If better-sqlite3 is not installed, install it.

Password hashing: bcrypt with cost factor 12 
(DSGVO-compliant minimum — never MD5, SHA1, 
or unsalted hashes).
Install bcryptjs (pure JS, no native deps).

Sessions: JWT stored in an httpOnly cookie 
(not localStorage). Use jsonwebtoken package.
Token expiry: 8 hours. Refresh on activity.

Email: nodemailer — configure via settings 
stored in the database (see Mailserver section).
For now, log emails to console if no SMTP 
is configured (graceful fallback).

DATABASE SCHEMA

Create a SQLite database file at:
  server/admin.db

Tables:

TABLE users:
  id            INTEGER PRIMARY KEY AUTOINCREMENT
  email         TEXT UNIQUE NOT NULL
  username      TEXT UNIQUE NOT NULL
  password_hash TEXT NOT NULL        -- bcrypt
  first_name    TEXT NOT NULL
  last_name     TEXT NOT NULL
  role          TEXT NOT NULL        -- 'admin' | 'user_extended' | 'user'
  is_active     BOOLEAN DEFAULT 0   -- 0 until first login complete
  must_change_pw BOOLEAN DEFAULT 1  -- force pw change on first login
  created_at    DATETIME DEFAULT CURRENT_TIMESTAMP
  created_by    INTEGER REFERENCES users(id)
  last_login    DATETIME

TABLE verification_codes:
  id          INTEGER PRIMARY KEY AUTOINCREMENT
  user_id     INTEGER NOT NULL REFERENCES users(id)
  code        TEXT NOT NULL          -- 6-digit numeric
  purpose     TEXT NOT NULL          -- 'email_change' | 'data_change'
  new_value   TEXT                   -- new email if purpose=email_change
  expires_at  DATETIME NOT NULL      -- 15 minutes from creation
  used        BOOLEAN DEFAULT 0
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP

TABLE settings:
  key         TEXT PRIMARY KEY
  value       TEXT NOT NULL
  updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP

BACKEND API ROUTES

Add these routes to the Express/Node backend.
Prefix all with /api/admin/

AUTH:
  POST /api/admin/login
    Body: { username, password }
    Returns: sets httpOnly JWT cookie
    Response: { user: { id, email, username, 
      firstName, lastName, role } }

  POST /api/admin/logout
    Clears the JWT cookie

  GET /api/admin/me
    Returns current user from JWT cookie
    Returns 401 if not authenticated

USER MANAGEMENT (admin only):
  GET /api/admin/users
    Returns all users (id, email, username, 
    firstName, lastName, role, isActive, 
    createdAt, lastLogin)

  POST /api/admin/users
    Body: { firstName, lastName, email }
    Creates new user with:
      - username = first initial + lastName 
        (e.g. "sHaase"), made unique by 
        appending a number if needed
      - temporary password: random 12-char 
        string (upper+lower+digit+special)
      - role: 'user' (always — admin cannot 
        directly create admin)
      - must_change_pw: true
      - is_active: false (activated on 
        first login)
    Sends welcome email with temp credentials.
    Returns the created user (without password).

  PATCH /api/admin/users/:id/role
    Admin only. Body: { role }
    Cannot set role to 'admin' if user's 
    is_active is false.
    Cannot demote yourself if you are the 
    only admin.
    Returns updated user.

  DELETE /api/admin/users/:id
    Admin only. Cannot delete yourself.

USER PROFILE (all authenticated users):
  GET /api/admin/profile
    Returns own user data

  PATCH /api/admin/profile
    Body: { firstName?, lastName?, 
            newUsername?, newEmail?, 
            currentPassword?, newPassword? }
    
    For name changes: apply immediately, 
    send confirmation email (info only).
    
    For username/password changes: validate 
    currentPassword first. Apply if correct.
    Send confirmation email.
    
    For email change: 
      1. Generate 6-digit code
      2. Store in verification_codes 
         (purpose='email_change', 
          new_value=newEmail)
      3. Send code to the CURRENT email address
      4. Return { requiresVerification: true }
      5. User must call /verify-code with the code

  POST /api/admin/verify-code
    Body: { code, purpose }
    Validates the code (not expired, not used,
    matches user).
    If purpose='email_change': update email.
    Marks code as used.

FIRST LOGIN (force password change):
  POST /api/admin/set-initial-password
    Body: { username, tempPassword, 
            newPassword, confirmPassword }
    Only works when must_change_pw = true.
    Validates temp password, validates new 
    password policy (see below), sets 
    is_active=true, must_change_pw=false.

PASSWORD POLICY (enforce on all password sets):
  - Minimum 10 characters
  - At least one uppercase letter (A-Z)
  - At least one lowercase letter (a-z)
  - At least one digit (0-9)
  - At least one special character 
    (!@#$%^&*()_+-=[]{}|;':",.<>?/~`)
  Return clear error messages for each 
  unmet requirement.

SECURITY REQUIREMENTS

- All /api/admin/* routes (except /login 
  and /set-initial-password) require a 
  valid JWT cookie
- JWT secret: read from environment variable 
  ADMIN_JWT_SECRET. Generate a random 64-char 
  hex string as default if not set, but log 
  a warning to use a real env var in production
- Rate limit /api/admin/login: 
  max 5 attempts per 15 minutes per IP
- Never return password_hash in any API response
- Verification codes: 6 random digits, 
  expire after 15 minutes, single-use
- Log all admin actions (user creation, 
  role changes, deletion) to console with 
  timestamp and actor
